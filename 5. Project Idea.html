Description (500 words) - Detailed description of the features of the product or service.

The purpose of the program will be to monitor the wireless network to identify new (unrecognised) devices. Wireless networks are susceptible to compromise due to the nature communication used, with most typical home wireless networks using WPA2 Pre-Shared Key (PSK). If an attacker is in range of these networks then they may be able to compromise the integrity of the network via a range of attacks. By identifying devices by their MAC (Media Access Control) address and mapping them to the known (trusted) user devices, it would be possible to limit the possibility of compromise while increasing visibility of network usage. Any unknown devices that appear on the network could then be quarantined until such time as they could be verified. It is important to note that this solution is intended for use on typical small home networks and the solution would be unsuitable for medium/large scale networks (e.g. corporate) as the manual verification would become too great an overhead. A Raspberry Pi or similar device would be sufficient to act as an 'always on' proxy and DHCP server (controller) which could also host the logic (e.g. bash/Python script) required to process and interrogate the devices joining the network. Alerts relating to new unrecognised hosts joining the networks could be sent via:

- Integration with a suitable SMS service API (Application Programming Interface) to a specified phone number (i.e. network owner/admin).
- Triggered communication to a mobile application controlled by the network owner/admin. This communication could be 2-way, meaning that logic could be included allowing the mobile application to communicate directly to the controller (Raspberry Pi logic) allowing actions to be taken (i.e. approve/quarantine devices).
- Local hardware such as physical LED lighting/screens or speakers attached to the Raspberry Pi.
- IOT LED lighting/screens or speakers reachable from the Raspberry Pi.

Two possible attack vectors which can follow the compromise of a wireless home network are:
1. Using the access to launch attacks on targets via the internet
2. Attacking clients on the local network
A combination of access-control technologies and defences will be required to mitigate these attacks.
The solution should be completely managed from the Raspberry Pi, with limited configuration required on the router/gateway/access point itself. This is a deliberate decision in order to avoid compatibility issues brands/models of network devices.
